## 背景与目标

- 当前后台为单页 Tab 切换（[Admin.tsx](file:///home/zl/work/workspaces/phowson/pages/Admin.tsx)），菜单按权限动态插入；整体可用但信息架构与“个人/管理员”业务边界不清。
- 目标：把后台管理中心重构为“个人中心 + 管理后台”两大域，统一交互与设计规范，在移动端可用、可扩展、性能可控，并满足导出与无障碍要求。

## 现状基线（便于后续开发对齐）

- 路由：`/admin` 入口在 [App.tsx](file:///home/zl/work/workspaces/phowson/App.tsx)；上传页路由为 `/upload`。
- 布局：后台目前在 [Admin.tsx](file:///home/zl/work/workspaces/phowson/pages/Admin.tsx) 内实现侧边栏（含移动端抽屉/遮罩）与 Tab 内容切换。
- 权限：后端 `requireAdmin()` 在 [rbac.mjs](file:///home/zl/work/workspaces/phowson/server/plugins/rbac.mjs)；Admin API 在 [admin.mjs](file:///home/zl/work/workspaces/phowson/server/routes/admin.mjs)，用户管理 API 在 [users.mjs](file:///home/zl/work/workspaces/phowson/server/routes/users.mjs)。

## 角色与权限模型

- 成员（member）：可访问“个人菜单区域”全部功能。
- 管理员（admin）：在 member 权限基础上可访问“管理员菜单区域”。
- 访问控制：

  - 前端：基于 `currentUser.role` 控制菜单与路由守卫。
  - 后端：管理员域所有接口必须 `requireAdmin()`；个人域接口至少 `requireMember()` 并限制仅访问自己的数据。

## 信息架构（IA）与路由建议

- 说明：为满足“统一分页/导出/可扩展”，建议把当前单页 Tab 迁移为“/admin 下的嵌套路由 + 统一 Layout”，避免一个巨型组件难维护。
- 后台根：`/admin`

  - 顶部固定：上传新照片（跳转 `/upload`）

    - 我的资料：`/admin/me/profile`
  - 下方区域（管理员，仅 admin 可见）：

    - 全部照片管理：`/admin/manage/photos`

## 一、整体布局重构（需求）

### 1) 左侧导航两大区域

- 左侧导航分为：

  - “个人相关菜单”与“管理员菜单”两块
  - 两块之间需有明显分隔（分隔线/分组标题/背景差异均可，保持统一规范）
- 移动端：沿用当前侧边栏抽屉 + 遮罩模式，保证可用性。

### 2) 顶部固定上传按钮

- 常驻在左侧导航顶部，视觉优先级最高。
- 点击跳转至现有上传新照片页面：`/upload`（已存在路由）。

### 3) 视觉规范

- 个人区与管理员区：

  - 分组标题（中文）、分隔线、间距一致。
  - 当前项高亮与 hover、focus 状态一致。

## 二、个人菜单区域（需求）

### 2.1 我的相册（相册视图）

**目标定位**

- 以“相册视图”呈现当前用户上传的照片，突出回顾与成长记录。

**页面结构**

- 顶部：页面标题 + 简短文案（强调“视觉捕捉历程与成长习惯”）
- 核心板块：

  - 那年今天：展示历史上同月同日的照片（按年分组或时间线）
  - 全部照片：网格相册（缩略图优先）
- 筛选区（多维度组合筛选）：

  - 月份筛选器：按月份/月份区间
  - 标签筛选器：多选
  - 相机型号筛选器：多选（来源于 EXIF）
- 列表：统一分页组件（见通用要求）或大数据量下的分页加载。

**交互与跳转**

- 从首页点击用户名称可直达该页（默认进入“我的相册”）。
- 点击任一照片可跳转到详情页（现有 `/photo/:id`）。

**数据需求**

- 需要支持按：月份、标签、相机型号、关键词（可选）筛选。
- “那年今天”需要按当天（月-日）查询历史数据。

### 2.2 我的统计分析

**模块清单**

1. 本月活跃度（圆环图）

- 展示维度：24 小时（0-23）上传密度分布
- 交互：hover 显示数值；键盘可聚焦逐段读取

2. 每日目标

- 用户可设置每日上传目标数量（整数、范围限制、可关闭目标）
- 展示：今日完成进度（进度条/环形进度）

3. 上传量统计

- 展示上传量最高的具体日期
- 展示环比：

  - 同比/环比口径明确（默认：与上一周期对比；周期可定义为“上月/上周/昨日”之一，建议从“与上月同比”开始）

4. 活跃历史（年度热力图）

- 年度维度可切换年份（默认当前年）
- 直观看到全年活跃分布

5. 最近活动

- 最近 3 天上传记录列表

6. 数据导出（所有图表/模块）

- 导出形式：

  - 数据导出：CSV/JSON
  - 图表导出：PNG（可利用现有 `html2canvas` 依赖）

### 2.3 我的上传历史

**展示形态**

- 时间线：按日期分组（天粒度），每组内展示当日照片缩略图与基础信息。

**筛选与搜索**

- 日期选择器：精确到日
- 关键词搜索：标题/描述/标签（至少支持其中两项）

**跳转**

- 每张照片可点击进入详情页。

**主题文案**

- 统一页面主题：回顾每一次快门瞬间与足迹。

### 2.4 我的资料

**个人信息管理**

- 头像上传/更换
- 显示名称修改
- 角色信息展示（只读）

**账户安全**

- 修改密码：

  - 强度验证（长度、复杂度、常见弱口令拦截至少其一）
  - 原密码校验

**个性化设置**

- 每日上传目标设定（与“我的统计分析-每日目标”共用数据源）

**系统操作**

- 退出登录

**保存与反馈**

- 所有修改：实时保存（字段级或表单级自动保存均可，但需统一交互）
- 反馈：统一 toast 成功/失败提示；需要二次确认的操作用确认对话框（例如：退出登录、重置设置）。

## 三、管理员菜单区域（需求）

- 继承并优化现有照片管理功能（当前在 Admin 单页内）。

### 3.2 全部数据统计分析（全站级）

### 3.3 评论管理

  - 批量操作（批量通过/驳回/删除/标记）

### 3.4 用户管理

### 3.5 系统设置

- 整合现有网站设置（当前 `/admin/site-settings` 相关）
- 新增：

  - 配置备份：导出当前设置（JSON）
  - 配置恢复：导入 JSON 并校验后应用
  - 可选：备份历史列表（存 DB 或文件存储，二期）

## 四、通用技术与交互要求（需求）

### 4.1 统一分页组件

- 所有列表类页面必须使用统一分页组件：

  - 页码跳转
  - 每页数量选择
  - 显示总数/范围
  - 键盘可操作

### 4.2 样式统一（设计规范）

- 统一：色彩、字体、间距、圆角、阴影、交互反馈（hover/focus/active/disabled）。
- 产物：组件级规范（按钮、输入框、选择器、表格/卡片、空状态、加载状态）。

### 4.3 交互反馈

- 统一 toast 提示与确认对话框：

  - 成功/失败/警告/加载
  - 需要确认的高风险操作必须走确认对话框

### 4.4 移动端适配

- 所有页面响应式：手机/平板/桌面。
- 列表类在移动端可切换为卡片视图（项目既有策略可复用）。

### 4.5 性能优化

- 懒加载（路由级、图片级）
- 分页加载/服务端分页
- 大数据量下避免一次性渲染过多节点

### 4.6 无障碍（WCAG 2.1）

- 键盘导航：可聚焦、可操作、焦点可见
- 屏幕阅读：语义化标签、aria-label、图表可读替代信息
- 颜色对比度达标，非颜色单一表达状态

## 数据与接口（建议口径，便于拆分开发）

- 个人相册/历史：需要支持按用户、日期范围、标签、相机型号、关键词查询的分页接口。
- 统计分析：

  - 本月小时分布接口（按小时聚合）
  - 年度热力图接口（按日聚合，已有 activity/heatmap 思路可复用）
  - 最近 3 天接口（按天/按记录）
- 用户资料：头像上传、姓名修改、密码修改、目标设置。
- 管理端：

  - 评论批量操作接口
  - 用户管理：列表、编辑、禁用/启用、重置密码
  - 系统设置：备份/恢复接口

## 导出能力（统一规范）

- 数据导出：CSV/JSON，字段规范化（时间 ISO、中文表头可选）
- 图表导出：PNG（以可视区域为基准），并提供导出前预览/范围选择（可二期）
