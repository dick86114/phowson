# 运维指南 (Operations Guide)

## 1. 系统监控体系

### 1.1 基础资源监控
建议使用 Prometheus + Grafana 监控服务器资源：
- **CPU/Memory**: 告警阈值 80%。
- **Disk Usage**: 数据库磁盘使用率告警阈值 75%。
- **Network**: 监控带宽峰值与异常流量。

### 1.2 应用级监控
- **API 响应时间**: 监控 P95/P99 延迟，超过 1s 触发告警。
- **错误率**: 5xx 错误率超过 1% 触发告警。
- **AI 接口配额**: 监控 Gemini/OpenAI 调用次数，接近配额时预警。

### 1.3 日志管理
系统日志默认输出到标准输出 (stdout)，建议通过 ELK (Elasticsearch, Logstash, Kibana) 或 Loki 进行收集与分析。
- **Access Log**: 记录所有 HTTP 请求。
- **Error Log**: 记录系统异常堆栈。
- **Audit Log**: 记录敏感操作 (如删除照片、修改权限)。

## 2. 备份与恢复

### 2.1 数据库备份
- **策略**: 每日全量备份，每小时增量备份 (WAL)。
- **工具**: `pg_dump` 或云厂商提供的自动备份服务。
- **脚本示例**:
  ```bash
  pg_dump -U postgres phowson > /backup/phowson_$(date +%Y%m%d).sql
  ```

### 2.2 图片存储备份
- 若使用 S3: 开启版本控制 (Versioning) 与跨区域复制 (Replication)。
- 若使用本地存储: 定期同步 `uploads/` 目录到异地服务器。

### 2.3 灾难恢复演练
每季度进行一次数据恢复演练，验证备份文件的有效性与恢复耗时。

## 3. 故障处理预案

### 3.1 数据库连接耗尽
- **现象**: API 响应 500，日志提示 `Too many connections`。
- **处理**:
  1. 临时重启应用释放连接。
  2. 检查慢查询 SQL (`pg_stat_activity`)。
  3. 增加连接池大小或引入 PgBouncer。

### 3.2 存储空间不足
- **现象**: 上传失败，日志提示 `No space left on device`。
- **处理**:
  1. 清理临时文件 (`/tmp`, `node_modules/.cache`)。
  2. 扩容磁盘或迁移旧日志。
  3. 紧急清理 `uploads/` 下未被引用的孤儿文件。

### 3.3 AI 服务中断
- **现象**: 填单功能失效，返回超时或错误。
- **处理**:
  1. 切换备用 AI Provider (如从 Gemini 切换到 OpenAI)。
  2. 启用降级模式，允许用户手动填单。

## 4. 日常维护清单

- [ ] **每日**: 检查错误日志，确认无严重异常。
- [ ] **每周**: 检查磁盘空间与备份状态。
- [ ] **每月**: 更新系统补丁与依赖包。
- [ ] **每季**: 数据库性能分析与索引优化。
