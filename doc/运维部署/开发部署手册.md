# 开发部署手册 (Deployment Manual)

## 1. 开发环境搭建

### 1.1 前置依赖
- **Node.js**: v18+ (推荐 v20 LTS)
- **pnpm**: v8+ (`npm i -g pnpm`)
- **PostgreSQL**: v15+ (需安装 pgvector 插件)
- **Docker** (可选，用于快速启动 DB)

### 1.2 本地启动步骤
1. **克隆代码**:
   ```bash
   git clone <repo-url>
   cd phowson
   ```

2. **安装依赖**:
   ```bash
   pnpm install
   ```

3. **配置环境变量**:
   复制 `.env.example` 为 `.env`，并填入必要信息：
   ```ini
   DATABASE_URL=postgres://user:pass@localhost:5432/phowson
   AI_API_KEY=your_gemini_key
   ```

4. **启动数据库 (Docker)**:
   ```bash
   docker compose -f docker-compose-postgres.yml up -d
   ```

5. **运行迁移**:
   ```bash
   pnpm db:migrate
   ```

6. **启动开发服务**:
   ```bash
   pnpm dev
   ```
   前端地址: `http://localhost:5173`
   后端地址: `http://localhost:3000`

## 2. 生产环境部署

### 2.1 Docker Compose 部署 (推荐)
本项目提供了一键部署脚本 `deploy.sh`。

1. **准备服务器**: Ubuntu 22.04 LTS, 2GB+ RAM。
2. **配置环境**:
   在服务器根目录创建 `.env` 文件，填入生产环境配置。
3. **执行部署**:
   ```bash
   ./deploy.sh
   ```
   该脚本会自动：
   - 构建 `phowson` 镜像 (All-in-One)。
   - 启动 PostgreSQL 容器。
   - 启动 Phowson 容器 (包含 Nginx 和 Node.js)。

### 2.2 手动部署 (Node.js)
1. **构建前端**:
   ```bash
   pnpm build
   ```
   产物在 `dist/` 目录。

2. **启动后端**:
   可以使用 PM2 管理进程：
   ```bash
   pm2 start server/index.mjs --name phowson
   ```

3. **配置 Nginx**:
   将前端静态文件指向 `dist/`，并将 `/api` 请求转发到后端端口。

## 3. 回滚方案

### 3.1 代码回滚
若发布后发现严重 Bug，可快速回滚到上一个 Docker Tag：
```bash
# 修改 docker-compose.yml 中的 image tag
image: phowson:v1.0.0 # 回退版本
docker compose up -d
```

### 3.2 数据库回滚
若迁移脚本导致数据异常，可使用 `down` 迁移 (需自行编写) 或恢复备份：
```bash
# 恢复备份
cat backup.sql | docker exec -i phowson-db psql -U postgres phowson
```

## 4. 常见问题 (Troubleshooting)

- **Q: 启动时报错 `pgvector extension not found`?**
  - A: 确保使用的 Postgres 镜像包含 pgvector 插件 (如 `ankane/pgvector`)。
- **Q: 图片上传失败?**
  - A: 检查 `uploads/` 目录是否有写入权限，或 S3 配置是否正确。
- **Q: AI 接口无响应?**
  - A: 检查 API Key 是否过期，或服务器网络是否能访问 Google/OpenAI API。
