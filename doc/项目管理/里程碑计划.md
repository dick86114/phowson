## 2. 详细交付计划

### M1: 工程化基座 (2025-02-28)
- **交付内容**:
  - 模块化的后端路由结构
  - 统一的错误处理与日志系统
  - API 自动化测试集
- **风险点**: 旧代码耦合度高导致重构延期。
- **应对**: 优先保证核心接口兼容，非核心暂缓。

### M2: 核心安全与认证 (2025-03-07)
- **交付内容**:
  - 集成 Clerk/JWT 认证
  - 完整的用户与角色管理功能
  - 数据库权限字段迁移
- **风险点**: 数据迁移导致用户信息丢失。
- **应对**: 迁移前全量备份，编写可回滚脚本。

### M3: 高性能图片服务 (2025-03-15)
- **交付内容**:
  - 对象存储接入 (S3/MinIO)
  - 图片缩略图与原图分离
  - 前端渐进式加载组件
- **风险点**: 历史图片迁移耗时过长。
- **应对**: 采用后台异步迁移策略，新图片直接上云。

### M4: Beta 版本发布 (2025-03-31)
- **交付内容**:
  - 瀑布流布局
  - 批量上传与 AI 辅助
  - 移动端适配优化
- **风险点**: AI 接口成本超支或响应慢。
- **应对**: 增加缓存层，设置调用频率限制。

### M5: 正式上线 (2025-04-15)
- **交付内容**:
  - 习惯养成系统 (Streak/Heatmap)
  - 地图模式
  - 全套运维与用户文档
- **风险点**: 上线后出现严重并发问题。
- **应对**: 提前进行压力测试，准备扩容预案。
