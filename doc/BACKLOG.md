# Backlog（可执行任务清单）

本清单把 [ROADMAP.md](./ROADMAP.md) 的 Phase 目标拆成可实现、可验收的任务条目，便于逐个迭代交付。

约定

- 每条任务都给出：范围、验收标准、影响面。
- 标注的文件路径为当前仓库位置，可随着实现调整。

---

## Phase 0：稳定化与工程化

### P0-1 后端路由与模块化重构

- 范围：将后端路由拆分为 photos/users/stats/admin 模块，并统一注册入口。
- 验收：现有 API 路径不变；接口行为一致；日志与错误返回格式一致。
- 影响：后端 [server/app.mjs](file:///home/zl/work/workspaces/phowson/server/app.mjs)

### P0-2 统一错误模型与错误码

- 范围：定义 `{ code, message, requestId }` 错误响应；保证 4xx/5xx 语义明确。
- 验收：错误不会暴露敏感信息；前端能展示统一提示。
- 影响：后端全局 error handler；前端 [api.ts](file:///home/zl/work/workspaces/phowson/api.ts)

### P0-3 请求校验（Schema Validation）

- 范围：对上传、评论、点赞等请求体/参数做校验，拒绝非法输入。
- 验收：错误返回 400 并带明确 code；合法请求不受影响。
- 影响：后端路由定义

### P0-4 API 冒烟测试

- 范围：为 /health、/photos、/photos/:id、/like、/comment、/stats 写最小自动化测试。
- 验收：CI/本地一键运行，失败可定位；测试不依赖浏览器。
- 影响：新增 test 目录与脚本

## Phase 1：身份认证与 RBAC

### P1-1 统一角色枚举与用户模型

- 范围：前后端统一 `admin|family|guest`；修正大小写混用。
- 验收：同一用户在前后端角色一致；不再出现 ADMIN/GUEST 与小写混杂。
- 影响：前端 [useAuth.ts](file:///home/zl/work/workspaces/phowson/hooks/useAuth.ts)、后端 header 解析逻辑

### P1-2 后端 RBAC 中间件

- 范围：后端按角色拦截写操作与后台接口；拒绝越权访问。
- 验收：直接调用 API 也会被拦截；返回 401/403 合理。
- 影响：后端路由层

### P1-3 接入真实登录（Clerk 或自建 JWT）

- 范围：落地真实登录与 token 校验，替换模拟登录。
- 验收：用户可登录/退出；token 过期可处理；后端可识别用户身份。
- 影响：前端登录页、后端鉴权

### P1-4 用户管理与分类管理后端化

- 范围：新增 categories 表与 CRUD；users 管理与角色变更；后台页面改用真实接口。
- 验收：后台可新增/编辑/删除分类；可管理用户角色；刷新不丢失。
- 影响：后端 DB/路由、前端 [Admin.tsx](file:///home/zl/work/workspaces/phowson/pages/Admin.tsx)

### P2-1 对象存储选型与接入（S3/R2/MinIO 等）

- 范围：上传图片写入对象存储（S3/R2/MinIO 等 S3 兼容）；数据库仅保存 URL/变体信息。

- 范围：生成 thumb/medium/original；列表加载 thumb，详情加载 medium/original。
- 验收：首屏加载明显更快；网络面板可见按需请求；图片质量符合预期。
- 影响：后端处理任务、前端列表/详情页
